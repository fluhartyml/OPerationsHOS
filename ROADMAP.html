  42. Phase 18 — Inbox tab promotion

  43. · Promote the existing Dashboard Inbox section to its own tab at position 2.
  44. · Inbox view: list of store.inbox (unpinned, undated, not surfaced elsewhere).
  45. · No new data model changes — the orphan filter logic is already in OperatorStore.

  47. Phase 19 — Schedule rewrite

  48. · Replace four-section ScheduleView with a calendar-native agenda list (inspired by Apple Calendar's List view).
  49. · Segmented control at top: OPerationsHOS (default, editable) / All Calendars (read-only context).
  50. · OPerationsHOS calendar is the editable, first-class one — only it gets AlarmKit alarms, Lock Screen widgets / complications, and Dynamic Island.
  51. · All Calendars view is read-only, pulls from every calendar the user has granted access to.

  53. Phase 20 — EventKit two-way sync

  54. · NSCalendarsFullAccessUsageDescription + NSRemindersFullAccessUsageDescription in Info.plist.
  55. · Create dedicated OPerationsHOS calendar in user's account on first launch (and dedicated OPerationsHOS reminders list).
  56. · Outbound: OperatorItems with dueDate sync to the dedicated calendar; OperatorItems of type Task / Reminder sync to the dedicated reminders list.
  57. · Inbound: events/reminders created in Calendar.app or Reminders.app inside the dedicated containers appear in OPerationsHOS.
  58. · Per-record sync state: eventIdentifier + reminderIdentifier on OperatorItem.

  60. Phase 21 — Reminders tab

  61. · New tab modeled on Reminders.app: list of items with checkboxes, due dates, optional alarms, grouped by section.
  62. · Backed by EKReminder via the dedicated OPerationsHOS reminders list.
  63. · Two-way sync (per Phase 20) — checking an item completes it in Reminders.app too.

  65. Phase 22 — AlarmKit integration on Timer records

  66. · NSAlarmKitUsageDescription in Info.plist.
  67. · Starting an OPerationsHOS Timer registers an AlarmKit timer for the system Lock Screen / Dynamic Island / completion alarm.
  68. · New Widget Extension target hosts the countdown Live Activity (AlarmKit requires this for any countdown presentation, otherwise the system may dismiss alarms).
  69. · AlarmKit is OPerationsHOS-calendar-only; other-calendar events don't get alarm registration.
  70. · Reference: AlarmKit User's Manual (Bible Appendix C, written 2026-05-05).

  72. Phase 23 — Vault

  73. · Repurpose Vault tab from universal-catch-all to biometrically gated container.
  74. · LocalAuthentication framework: Face ID / Touch ID gate on tab entry. No encryption — just biometric access control.
  75. · Contains: Media records (the existing Media type, now Vault-only) + Secure Notes (new ItemType case).
  76. · Records inside Vault are excluded from Dashboard sections, Inbox, search results outside Vault, and exports outside Vault.
  77. · Removes the standalone Media tab.

  79. Phase 24 — Lock Screen widgets / complications

  80. · WidgetKit configurations for OPerationsHOS dashboard data on Lock Screen and Home Screen.
  81. · Live Activity for in-progress AlarmKit timers (via the Phase 22 widget extension target).
  82. · OPerationsHOS-calendar-only.
